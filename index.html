Primer

A small machine for translating a family book.

NK-Book started with a single hardback memoir scanned on a phone, page by page. The pipeline takes those photos, runs them through Claude's vision model for OCR, then translates the Korean in 3-page chunks with a sliding context window so voice and proper nouns stay consistent across the whole book.

Every chunk is then audited by 3 independent agents: one back-translates the English into Korean and compares against the original, one scores tone and register, and one reconciles the notes into a quality grade. Anything below threshold gets retranslated, up to 2 attempts, before it's flagged for human review.

How it works

3 layers, every chunk audited.

  1. i

    OCR with Claude vision

    Each page photo goes to Claude's vision API. Korean text comes out exactly as written, with paragraph breaks preserved. PDFs get rasterized at 300 dpi first.

  2. ii

    Translation with a sliding window

    Text splits into 3-page chunks. A glossary of proper nouns and recurring terms carries across chunks so the same name renders the same way on page 4 and page 240.

  3. iii

    Validation by 3 agents

    A back-translator catches semantic drift. A tone scorer grades formality, emotion, and rhythm. An editorial agent reconciles the notes and assigns a grade. Anything under the bar is retranslated automatically.

Tech stack

Built with a small kit.

Plain Python and the Anthropic SDK do the heavy lifting. No web framework, no database, no orchestration layer. Every step is resumable: if it crashes mid-run, re-running picks up where it left off.

  • Python 3.11
  • Claude API (vision + text)
  • Anthropic SDK
  • pdf2image
  • Pillow
